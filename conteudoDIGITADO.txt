arquivo.h define as definições das funções, mas nn as compila
arquivo.c possui apenas as funções a serem compiladas

OPERAÇÃO BASICA:

criação do TAD:
inserção de um novo elemento no TAD
remoção de um elemento do TAD
busca de um elemento no TAD
impressão do TAD
liberação da memória alocada para o TAD


CRIAÇÃO DE BIBLIOTECA:

<include> "ponto.h"

quando se tem 2 códigos .c, é necessario executar os dois códigos ao msm tempo, e o ponto.h só serve para modularizar o códigos

COMIT

para entrar em uma pasta:
cd <nome da pasta>

para sair da pasta e ir para uma superior:
cd ..

para excutar um programa:
ggc <nome do programa.c> -o <programa quem vai ser usado na saida> 

executar o arquivo na pasta nessa pasta:

./main

PONTEIRO QUE APONTA PARA UM PONTEIRO:

<asterisco><asterisco><variaveis>

não aponta mais para uma variavel, aponta para outro ponteiro e esse outro ponteiro aponta para a variavel

serve para manipular endereços de ponteiros

LISTA:

STACK:  local na memória que armazena dados temporários, guardando os valores dass funções e subprogramas, quando terminar a execução os valores saem da memoria

HEAP : é o local de alocação dinâmica da memória, definida pelo programador

lista inicializa como NULL, vazio.

COMO CHECAR O TAMANHO DA LISTA:

passa o ponteiro para ponteiro como parametro

se der algum problema com o ponteiro, como ele apontar para um espaço vazio, o código da um loop para garantir q a lista não está mnais vazia

tudo tem q estar na memória
